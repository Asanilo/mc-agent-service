# mc-agent-service API Reference

This document is the public API contract for `mc-agent-service`. It defines the REST, WebSocket, and MCP surfaces exposed by the service described in `SPEC.md`.

The service owns Mineflayer bots through isolated workers. API adapters never expose live Mineflayer objects; every request and response is validated JSON.

## Conventions

- Base HTTP URL: `http://host:port`
- WebSocket URL: `ws://host:port/ws`
- Timestamps: ISO 8601 UTC strings.
- IDs: opaque strings generated by the service unless explicitly supplied by the client.
- Request and response media type: `application/json`.
- Empty successful mutating responses still return a JSON body.
- Error response format is defined in [Error Handling](#4-error-handling).
- Authentication, when enabled, uses `X-API-Key` unless configured otherwise.

## 1. REST API

### POST /bots

Create a bot record and optionally connect it immediately.

**Request body schema**

```json
{
  "type": "object",
  "properties": {
    "bot": { "$ref": "#/components/schemas/BotConfig" },
    "connect": { "type": "boolean", "default": true }
  },
  "required": ["bot"],
  "additionalProperties": false
}
```

**Response body schema**

Status: `201 Created`

```json
{
  "type": "object",
  "properties": {
    "bot": { "$ref": "#/components/schemas/BotDetail" }
  },
  "required": ["bot"],
  "additionalProperties": false
}
```

**Error codes**

- `BOT_ALREADY_EXISTS`: requested bot ID already exists.
- `VALIDATION_FAILED`: request body does not match schema.
- `AUTH_REQUIRED`, `AUTH_INVALID`, `AUTH_FORBIDDEN`
- `RATE_LIMITED`
- `SERVICE_UNAVAILABLE`: worker could not be started.
- `INTERNAL_ERROR`

### POST /bots/{botId}/start

Start or reconnect a bot. If the bot is already running, this is idempotent.

Note: `forceReconnect` and `reason` are accepted in the request body but not yet wired to the bot lifecycle.

**Request body schema**

```json
{
  "type": "object",
  "properties": {
    "forceReconnect": { "type": "boolean", "default": false },
    "reason": { "type": "string", "maxLength": 512 }
  },
  "additionalProperties": false
}
```

**Response body schema**

Status: `202 Accepted`

```json
{
  "type": "object",
  "properties": {
    "bot": { "$ref": "#/components/schemas/BotSummary" },
    "accepted": { "type": "boolean" }
  },
  "required": ["bot", "accepted"],
  "additionalProperties": false
}
```

**Error codes**

- `BOT_NOT_FOUND`
- `BOT_DESTROYED`
- `BOT_START_FAILED`
- `VALIDATION_FAILED`
- `AUTH_REQUIRED`, `AUTH_INVALID`, `AUTH_FORBIDDEN`
- `RATE_LIMITED`
- `INTERNAL_ERROR`

```json
  "additionalProperties": false
```

```json
  "type": "object",
  "properties": {
    "bot": { "$ref": "#/components/schemas/BotSummary" },

      "type": "array",
```

**Request body schema**

```json

  "properties": {
```

## 2. WebSocket API

Connect to a specific bot:

```text
ws://host:port/ws?botId={botId}
```

Connect to all events:

```text
ws://host:port/ws
```

Authentication, when enabled, may be supplied with the configured API key header during the WebSocket upgrade or as `?apiKey=...` when header injection is unavailable.

```json
{
  "type": "object",

  "required": ["id", "ts", "type", "data"],

```


```json
{
  "type": "object",
  "properties": {
    "type": { "type": "string", "const": "subscription.changed" },

      "type": "object",
      "properties": {
        "subscribedBotIds": {
          "type": "array",
          "items": { "type": "string" }
        }
      },
      "required": ["subscribedBotIds"],
      "additionalProperties": false
    }
  },
  "required": ["type", "data"],
  "additionalProperties": false
}
```

#### unsubscribe

Remove a bot subscription from the connection.

```json
{
  "type": "object",
  "properties": {
    "type": { "type": "string", "const": "unsubscribe" },
    "botId": { "type": "string" }
  },
  "required": ["type", "botId"],
  "additionalProperties": false
}
```

Acknowledgement uses the same `subscription.changed` schema as `subscribe`.

## 5. Authentication

Authentication is optional and disabled by default.

Default config:

```json
{
  "auth": {
    "mode": "none"
  }
}
```

API key mode:

```json
{
  "auth": {
    "mode": "api-key",
    "header": "X-API-Key",
    "keyEnv": "MCAGENT_API_KEY"
  }
}
```

Clients send:

```http
X-API-Key: <key>
```

The API key value is read from the `MCAGENT_API_KEY` environment variable by default. The header name can be changed with `MC_AGENT_API_KEY_HEADER`.

Bearer mode is also supported by the architecture:

```json
{
  "auth": {
    "mode": "bearer",
    "tokenEnv": "MC_AGENT_AUTH_TOKEN"
  }
}
```

Clients send:

```http
Authorization: Bearer <token>
```

Secrets are never returned in API responses. Bot configuration responses redact `password` values. `GET /bots/{botId}` currently returns an empty `config` object; `POST /bots` returns the full config (redaction is not yet applied to the create response).

## 6. Rate Limiting

Currently only chat rate limiting is implemented (`createChatRateLimitMiddleware`). General REST/WS/MCP rate limiting middlewares are not yet wired.

Rate limiting is enabled per adapter and can be disabled for trusted local deployments. Limits are keyed by API key when authenticated, otherwise by remote address.

Default limits:

| Scope | Default |
| --- | ---: |
| REST read requests | 600 requests/minute |
| REST mutating requests | 120 requests/minute |
| Skill job creation | 60 requests/minute |
| Chat sends | 20 messages/minute per bot |
| WebSocket client messages | 120 messages/minute |
| WebSocket connections | 20 concurrent connections per key/address |
| MCP tool calls | 120 calls/minute |
| MCP mutating tool calls | 60 calls/minute |

Configuration shape:

```json
{
  "rateLimit": {
    "enabled": true,
    "rest": {
      "readPerMinute": 600,
      "mutatePerMinute": 120
    },
    "jobs": {
      "createPerMinute": 60
    },
    "chat": {
      "messagesPerMinutePerBot": 20
    },
    "websocket": {
      "clientMessagesPerMinute": 120,
      "maxConnections": 20
    },
    "mcp": {
      "toolCallsPerMinute": 120,
      "mutatingToolCallsPerMinute": 60
    }
  }
}
```

Rate-limited REST responses return:

```json
{
  "error": {
    "code": "RATE_LIMITED",
    "message": "Rate limit exceeded.",
    "details": {
      "retryAfterMs": 1000,
      "scope": "rest.mutate"
    }
  }
}
```

HTTP status: `429 Too Many Requests`.

When possible, REST responses include:

```http
Retry-After: 1
X-RateLimit-Limit: 120
X-RateLimit-Remaining: 0
X-RateLimit-Reset: 2026-06-20T12:00:00.000Z
```
